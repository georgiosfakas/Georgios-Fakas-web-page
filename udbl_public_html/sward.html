SWARD - Semantic Web Abridged Relational Databases


This work is funded by the IST project EGovBus.

The SWARD system provides semantic web representations using RDF and RDF Schema of data stored in conventional relational databases For a given relational database the user defines a wrapper that exports chosen parts of the database as a view in terms of the RDF meta-data model. The exported view can be queried using either SQL or one of the RDF-oriented query languages SparQL and RDQL. The RDF-view of a relational database is automatically generated from two simple mapping tables in SWARD that, for a given ontology, map tables to classes and columns to properties.

Using SWARD, we are developing a virtual repository system that enables scalable SQL queries to RDF views of large relational databases storing Government documents and life event data. This provides very flexible searchable views of wrapped relational databases. Using the RDF-model both relational meta-data (schema) and stored data can be queried. By providing meta-data, queries, and results using basic semantic web representations such as RDF-Schema the system provides scalable access to large eGovernment repositories from advanced tools that need high-level semantic descriptions of integrated information.

An RDF view of a relational database is defined as a large disjunctive query. Optimization of queries to large disjunctive views is challenging and makes it critical to optimize not only data access time but also the time to perform the query optimization itself. We have developed novel query optimization techniques based on query rewrites and compile time evaluation of subexpressions that enables execution of real-world queries to RDF views of relational databases.
